# 📊 סטטוס פיצ’רים בפרויקט (Feature Status)

מסמך זה מרכז את כל היכולות המתוכננות למערכת "סדר יום דיגיטלי", ומציין מה כבר קיים ומה עדיין בפיתוח.

## ✅ פיצ’רים קיימים (Implemented)

### 📅 לוח סדר יום (Daily Schedule)

- [x] **תצוגה ויזואלית**: רשימת משימות עם כרטיסיות גדולות, טקסט ותמונה/איקון.
- [x] **אינטראקטיביות**:
  - סימון משימה כבוצעה (V) בלחיצה.
  - אפקט ויזואלי (Gray out) למשימות שבוצעו.
- [x] **גרירה ושחרור (Drag & Drop)**: אפשרות לסידור מחדש של המשימות (פעיל במצב עריכה בלבד).
- [x] **מצב עריכה (Edit Mode)**:
  - מוגן/מופרד ע"י כפתור נעילה (Lock/Unlock).
  - הוספת משימה חדשה (מודל עם בחירת תמונה ושם).
  - מחיקת משימות.
  - יצירת רשימות חדשות, החלפת רשימות, ומחיקת רשימות.
- [x] **שמירת נתונים (Persistence)**: שמירה אוטומטית ומקומית (LocalStorage) של הרשימות והסטטוס בדפדפן.

### 🎉 משוב וחיזוקים (Feedback)

- [x] **חיווי סיום משימה**: השמעת צליל ("Ding") בעת סימון משימה.
- [x] **חגיגה (Celebration)**: הצגת אנימציית קונפטי (Confetti) במרכז המסך בעת סיום מוצלח.

### ⚙️ תשתית (Infrastructure)

- [x] **פרופיל משתמש (בסיסי)**: תשתית State וממשק (UI) להצגת שם ואווטאר של המשתמש הנוכחי בכותרת.
- [x] **Localization**: ממשק מלא בעברית (RTL).

---

## 🚧 בפיתוח / מתוכנן (Planned / In Progress)

### 🔊 משוב קולי מתקדם

- [ ] **חיזוקים רנדומליים**: החלפת הצליל הקבוע ("Ding") בבנק של משפטי חיזוק מוקלטים ("כל הכבוד!", "אתה אלוף!", "סיימת יפה!") שמושמעים באופן אקראי למניעת קיבעון.

### 👥 ריבוי משתמשים (User Management)

- [x] **החלפת משתמשים**: אפשרות למעבר מהיר בין פרופילים של ילדים שונים (למשל: תמר, יונתן, אריאל).
- [x] **שמירת העדפות לכל משתמש**: סדר יום נפרד והתקדמות אישית לכל פרופיל.
- [x] **פרופיל משתמש מתקדם**: סוג משתמש (בן/בת) לפנייה מגדרית מותאמת, צבע נושא אישי.

### 🗣️ תקשורת (Communication)

- [ ] **לוח תקשורת מהירה**: כפתורים צפים או אזור ייעודי למעבר מהיר להבעת צרכים בסיסיים ("אני רעב", "שירותים", "סיימתי") או קישור לאפליקציית Grid חיצונית.

### 🖥️ דאשבורד (Dashboard)

- [ ] **מצב "מסך טלוויזיה" (Passive Display)**: תצוגה ייעודית למסך גדול בבית, המשקפת בזמן אמת את הסטטוס של כל המשתמשים (מי נמצא באיזו פעילות כרגע) ללא צורך באינטראקציה.

### 📚 למידה (Learning Tools)

- [ ] **משחק קריאה**: מודול נפרד ללימוד קריאה בשיטה חזותית (התאמת תמונה לאות/מילה), כפי שתואר באפיון.

### ☁️ סינכרון (Sync)

- [ ] **גיבוי ענן**: סנכרון הנתונים מול שרת/מסד נתונים (במקום רק LocalStorage) כדי שאפשר יהיה לנהל את הלוח מהטלפון של ההורה ולראות את השינויים בטאבלט של הילד.
